¿Qué es el DOM?

El DOM (Document Object Model) es una representación estructurada en forma de árbol del contenido de un documento HTML. Proporciona una interfaz de programación para acceder y manipular dinámicamente los elementos de una página web.

¿Qué es un nodo?

En el DOM, un nodo es cada uno de los elementos en el árbol que representa la estructura del documento HTML. Los nodos pueden ser elementos, atributos, texto u otros tipos de contenido.

¿Cómo se accede a un elemento del DOM?

Para acceder a un elemento del DOM, se puede utilizar JavaScript y sus métodos proporcionados por el DOM. Algunas formas comunes de acceso son mediante el uso de los métodos getElementById, getElementsByClassName o querySelector.

¿Cómo se modifica un elemento del DOM?

Para modificar un elemento del DOM, se pueden utilizar métodos como innerHTML para cambiar su contenido, o los métodos para modificar atributos, como setAttribute o style, para cambiar los valores de los atributos o los estilos del elemento.

¿Cuál es la diferencia entre getElementby* y querySelector*?

La diferencia principal entre los métodos getElementby* y querySelector* radica en la flexibilidad y la forma de selección de los elementos. Los métodos getElementby* son más antiguos y permiten seleccionar elementos por su etiqueta, clase o nombre, pero solo devuelven una colección de elementos específicos. En cambio, los métodos querySelector* son más modernos y permiten seleccionar elementos utilizando selectores CSS más avanzados, como selectores de clase, id, atributos o incluso selectores complejos, y devuelven el primer elemento que cumple con el selector especificado.

¿Qué es un evento?

Un evento es una acción o suceso que ocurre en una página web, como hacer clic en un elemento, mover el ratón, cargar la página, entre otros. Los eventos permiten al usuario interactuar con la página y al desarrollador responder a esas interacciones ejecutando código JavaScript específico.

¿Qué es un listener?

Un listener (o "escuchador") es una función en JavaScript que se asocia a un elemento del DOM y espera la ocurrencia de un evento específico en ese elemento. El listener "escucha" o detecta el evento y ejecuta el código que se ha definido para responder a dicho evento.

¿Qué es un callback?

Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que cierto proceso o evento haya finalizado. En el contexto del DOM, los callbacks se utilizan frecuentemente para responder a eventos, donde la función callback se ejecuta cuando el evento ocurre.

¿Qué es un objeto de evento?

Un objeto de evento es un objeto que se crea y se pasa automáticamente como argumento a una función de manejo de eventos cuando ocurre un evento en JavaScript. Contiene información relevante sobre el evento que ocurrió, como el tipo de evento, el elemento al que se aplicó el evento y los detalles específicos del evento en sí.
